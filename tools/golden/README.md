# Golden-game OCR ground truth + scoring harness

A **vision-verified ground truth** for one complete Apex game and a harness to score/tune the
OCR killfeed pipeline against it. Turns "where does OCR break?" into hard recall/precision numbers,
and gives a fast edit→validate loop for parser/OCR fixes (no live capture needed).

## The golden game

Sang's **win** on VOD `twitch.tv/videos/2819348747` @ offset `01:00:00` (~858s). Claude vision
(opus, 1 read/sec) transcribed the whole killfeed → **53 distinct eliminations** (of the ≥57
floor; the rest are knock-only/feed-gap). This is the ground truth OCR is measured against.

Current at last run: **recall 90%, precision 85%, attacker-name 93%** (OCR pipeline vs vision;
golden denominator = 51 after two crop-verified vision-misread corrections). Path from the 77%
baseline is decomposed in `score_ocr.py` and the beads; the biggest single lever (respawn-gap in
the collapse) is a HARNESS measurement and is reachable-pending the same fix in production db_log.

## data/

| file | what | committed? |
|---|---|---|
| `vision_reads.jsonl` | Claude-vision reads (the ground truth — cost ~$3 to regenerate) | ✅ yes |
| `vod_capture/reads.jsonl` | raw OCR reads for the game (one per killfeed crop) | ✅ yes |
| `vod_capture/distinct_eliminations.jsonl` | OCR pipeline's collapsed kills (regenerated by parse_game) | ✅ yes |
| `vod_capture/crops/` | OCR narrow-zone color crops (for OCR/skull re-validation) | ⛔ gitignored (29 MB) |
| `kf_crops/` | wide killfeed crops (for a fresh vision read) | ⛔ gitignored (164 MB) |

The crops are gitignored (bulky) but regenerable — see below. The two JSONLs that cost time/money
(`vision_reads.jsonl`, `reads.jsonl`) are committed.

## Workflows

**Validate a PARSER fix** (parsers.py — names, markers, event types) — seconds, no re-OCR:
```
python tools/golden/parse_game.py     # reads.jsonl -> distinct_eliminations.jsonl
python tools/golden/score_ocr.py      # scorecard vs vision golden
```

**Validate an OCR / skull / gap fix** (ocr.py) — re-OCRs the saved crops (~7 min, no VOD decode):
```
python tools/golden/reocr_crops.py    # re-OCR crops -> reads.jsonl (backs up reads.jsonl.orig once)
python tools/golden/parse_game.py
python tools/golden/score_ocr.py
```

**Debug why a skull is missed** on one crop:
```
python tools/golden/skull_diag.py tools/golden/data/vod_capture/crops/00756.1_r0.png
```

**Regenerate the vision ground truth** (costs ~$3, needs ANTHROPIC_API_KEY):
```
python tools/golden/kf_crops.py  <vod_url> 01:00:00 860   # extract wide killfeed crops -> data/kf_crops/
python tools/golden/vision_read.py run 8                   # Claude vision -> data/vision_reads.jsonl
```

**Capture a different golden game** (find a win, then capture its crops+reads):
```
python tools/golden/peek_frame.py     <vod_url> 01:14:00 ...   # eyeball frames at offsets
python tools/golden/find_game_arc.py  <vod_url> 01:00:00 1600  # squads-left arc -> find a full game / win
python tools/golden/capture_game.py   <vod_url> <offset> <dur> # OCR crops + reads.jsonl + squads timeline
```

All scripts resolve `data/` relative to this folder, so run them from the repo root with
`PYTHONPATH` pointing at the repo (so they can import `ocr`, `parsers`, `database`, etc.).

## score_ocr.py output

- **recall** = golden kills OCR captured; **precision** = OCR kills that are real; **attacker** = name accuracy on matched.
- Each miss is classified **NEVER CAPTURED** (detection/zone — rare) vs **LOST DOWNSTREAM**
  (in the raw reads but dropped by parse/collapse — the cheap-fix bucket). That partition is the tuning map.
